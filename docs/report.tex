\begin{callout}
\textbf{Abstract.}
We describe a proof-of-concept ML pipeline for detecting and counting adult
    \textit{Lycorma delicatula} a.k.a Spotted Lanternfly(SLF) in pheromone sticky trap
photographs. The challenge is to propose a model that has reasonable accuracy 
    with no labeled training dataset, and limited to deployment on a CPU. These restrictions 
    prevent the use of common state-of-the-art 
    techniques.  Our proposed solution combines a HOG, LBP and Random Forest classifier trained from synthetic
ground truth. A preliminary evaluations shows that our approach achieves 85\% precision, 77\% recall and 81\% F1 with no GPU and no manually labeled images. We conclude by a tentative design for a production
deployment architecture that incorporates a feedback loop that uses uncertain in-production
detections to continuously grow the training dataset with minimal expert effort.
\end{callout}


\section{Problem Definition}

The goal of this project to detect and count adult \textit{Lycorma delicatula}, refered to as SLF, in 
pheromone sticky trap photographs. A SLF is an insect with wings. These insects get caught on a trap, 
which has a sticky surface that attracts various insects, fallen debris, leaves etc. The photograph can be 
tainted by blur, light etc. making this task challanging. A trap with caught insects is shown in 
Figure~\ref{fig:example}. We will use this image as a running example throughout the report.

\subsection{Scope}
 This project is only a proof-of-concept specifically designed to address the cold-start problem of having no existing labeled training dataset.  The project scope is explicitly limited to a working baseline system that demonstrates feasibility rather than a 
production-ready monitoring platform, with all performance validation conducted on synthetic images and clear acknowledgment 
that real-world performance requires validation on field data.

\section{Proposed Solution}
In this section we describe the proposed solution. The challenge is to find 
a machine learning approach that will give reasonable accuracy (F1\footnote{F1 is calculated as 2x(precision*recall/precision+recall)} $\approx 0.8$) 
given the constraints. Most state-of-the-art zero shot
approaches (e.g., zero-shot deep learning~\cite{clip}) require a GPU and deep learning require training data (~\cite{deep}). We therefore 
propose a technique that utilizes an heuristic approximation combined with classical machine learning techniques trained on 
synthetic data.

\subsection{Pre-processing}
The preprocessing stage (Stage 1) normalizes raw trap 
images to ensure consistent input for the detection pipeline, regardless of varying field photography conditions. First we 
use the \emph{CLAHE Enhancement} to remove uneven 
lighting (bright spots, shadows, overcast vs sunny). CLAHE divides image into 8×8 grid tiles, and equalizes histogram in each tile separately. This makes local contrast consistent across the image, 
regardless of global lighting. Next, we employ \emph{glare removal}. We find pixels brighter than 97\% of the image and use surrounding 
pixel colors to "paint over" the glare spots. This eliminates false edges that would confuse the region proposer. Lastly, we resize to standard resolution. Since field photos may vary 
wildly (e.g., 480×640, 1920×1080, 4032×3024, etc.), we standardize them to 1024×1024 for consistent processing. Thus 
all downstream size thresholds have consistent meaning. This ensures that the same insect photographed under different 
conditions produces similar-looking input to the detection stages, making the pipeline robust across varying field conditions.The 
preprocessing is deterministic and fast (<100ms on CPU), making it suitable for real-time field deployment.

\subsection{Region Proposal}
The region proposal stage serves as the critical bridge between processing the entire image and focusing computational 
resources on specific areas that might contain insects. Without this stage, the detection pipeline would face an intractable 
search problem: examining every possible location and scale within a 1024×1024 image would require evaluating millions of 
potential bounding boxes, making real-time detection impossible.

The fundamental challenge is that insects represent a tiny fraction of the total image area. A typical trap photograph shows 
mostly yellow sticky board background, with perhaps three to six small insects scattered across the surface. Rather than running 
the expensive descriptor classifier on every possible rectangular region, the proposal stage uses fast classical computer vision 
techniques to identify a manageable number of candidate locations that warrant closer examination.

The implementation combines two complementary detection methods to ensure comprehensive coverage. MSER 
(Maximally Stable Extremal Regions) excels at finding compact, blob-like structures that remain consistent across 
multiple intensity thresholds. This technique is particularly effective at detecting insect bodies, which typically 
appear as coherent dark or light regions against the trap background. MSER works by progressively thresholding the image 
at different intensity levels and identifying regions that persist across multiple thresholds, indicating stable, 
well-defined objects rather than noise or texture variations.

The second method employs Canny edge detection followed by contour analysis to capture objects defined by strong boundaries rather 
than uniform intensity. This approach complements MSER by detecting insects with prominent wing edges or clear silhouettes that might 
not register as stable intensity regions. Wing outlines, body contours, and other edge-defined features that MSER might miss are 
captured through this boundary-based approach.

Both methods produce initial candidate regions that undergo size filtering to eliminate obviously incorrect detections. Regions 
smaller than 300 square pixels likely represent noise, debris fragments, or imaging artifacts, while regions larger than 18,000 square 
pixels probably encompass multiple objects or large background areas rather than individual insects. These size thresholds correspond roughly to 3-30 millimeter objects at typical trap photography distances.

The candidates from both detection methods are then merged and processed through non-maximum suppression using intersection-over-union 
scoring. This step eliminates duplicate detections where MSER and Canny contours have identified overlapping regions of the same object, ensuring that each potential insect is represented by a single bounding box rather than multiple overlapping proposals.

The output of this stage is typically 15-50 bounding boxes per image, representing a dramatic reduction from the millions of possible locations while maintaining high confidence that genuine insects have been captured. The region proposer achieves very high recall, meaning it rarely misses actual insects, though it also generates false positives that represent debris, shadows, or other non-insect features. This high-recall, moderate-precision trade-off is appropriate for a proposal stage since downstream processing will refine the candidate list.
The computational efficiency gained through region proposal is substantial. Instead of requiring the descriptor classifier to 
extract HOG and LBP features from thousands of sliding window positions across multiple scales, the pipeline only needs to process 
a few dozen carefully selected candidates. This focus enables real-time processing on CPU-only hardware while ensuring that the subsequent classification stages receive a manageable and well-curated set of potential targets for detailed analysis.

\subsection{Heuristic Filter}
The heuristic filter stage (Stage 3a) serves as a fast, rule-based pre-screening step that evaluates candidate regions using expert knowledge about Spotted Lanternfly appearance. This stage receives the bounding boxes identified by the region proposer and applies three simple tests to determine which candidates are worth passing to the computationally expensive descriptor classifier.
The first test examines color compatibility by converting each candidate patch to HSV color space and counting how many pixels fall within the known color ranges for SLF adults. Since SLF forewings are typically red-brown and their bodies are greyish, the filter looks for patches containing a meaningful proportion of pixels in these hue ranges. Patches that are mostly bright yellow trap board background or dark shadows receive low scores, while patches with substantial reddish-brown or grey content score higher.
The second evaluation focuses on shape through aspect ratio analysis. Adult SLF at rest have their wings folded and appear wider than they are tall, typically with width-to-height ratios between 1.2 and 5.0. The filter gives perfect scores to candidates within this range and zero scores to regions that are too square or too elongated, as these shapes are inconsistent with SLF morphology.
The third assessment measures solidity, which quantifies how compact and filled a region appears by comparing the actual contour area to its convex hull area. SLF adults present as solid, coherent shapes, whereas common debris like twigs, leaf fragments, or plant material tends to be spindly and fragmented. The filter favors candidates with high solidity scores that suggest a compact insect body rather than irregular debris.
These three scores are combined into a single confidence measure, and only candidates exceeding a minimum threshold are passed forward to the descriptor classifier. This filtering approach is called heuristic because it relies on hand-crafted rules derived from domain expertise rather than learned patterns from data. The thresholds and color ranges were determined through manual inspection of SLF reference images and iterative testing on synthetic trap images.
The heuristic filter achieves very high recall at the cost of modest precision, meaning it successfully identifies nearly all actual SLF while also passing through many false candidates. This trade-off is intentional since the primary goal is computational efficiency rather than final accuracy. By eliminating obviously incorrect candidates in microseconds per region, the filter reduces the workload for the downstream classifier from typically 50-80 candidates per image down to 15-20 promising regions. This preprocessing step makes the overall pipeline several times faster while ensuring that genuine insects are rarely discarded prematurely.

\subsection{Descriptor classifier: HOG + LBP + RF}
The descriptor-based classifer stage (Stage 3b) represents a deliberate choice of classical
computer vision methods over modern deep learning. While CNN-based detectors
(YOLOv8, DETR) or vision transformers would likely achieve superior performance
given sufficient training data, the HOG\,+\,LBP\,+\,RF combination provides an
intermediate solution that works with minimal labeling effort, no GPU
infrastructure, and straightforward deployment.

\textbf{HOG} (64\,$\times$\,128\,px, 9 orientations, 8\,$\times$\,8 cells,
2\,$\times$\,2 block normalisation) captures wing outline shape and body
silhouette \cite{dalal2005hog}.
\textbf{LBP} (radius 3, 24 points, uniform mapping) captures surface texture;
the spotted forewing pattern produces a distinctly different histogram from
plain glue board \cite{ojala2002lbp}.
\textbf{HSV histogram} (18\,+\,8\,+\,8 bins) and \textbf{shape statistics}
(aspect ratio, solidity, eccentricity, 7 Hu moments) complete the
${\sim}600$-dimensional feature vector.
Features are normalised with \code{StandardScaler} and classified by a
\textbf{Random Forest} (200 trees, balanced class weights, max depth 12).

This combination was common in computer vision applications circa 2005--2015,
before deep learning became dominant. Modern insect detection systems typically
use end-to-end CNN architectures \cite{saleem2019plant}, but require
substantially more labeled training data (thousands vs.\ hundreds of examples)
and GPU infrastructure for practical deployment.

Instead of needing humans to manually draw bounding boxes around real SLF in real trap photos, the system generates its own training data through the synthetic image generator.


The synthetic generator procedurally creates fake trap images by rendering SLF-like shapes on simulated trap board backgrounds. Since these images are generated by code, the system automatically knows exactly where each synthetic insect is located - it has perfect ground truth "for free" because it placed the insects there in the first place.
When the synthetic generator creates an image with three SLF adults at coordinates (150,200), (400,350), and (800,450), those locations become the ground truth labels. No human annotation is required because the algorithm that created the image also records exactly what it created and where.
The classifier training then works by running the region proposer and heuristic filter on these synthetic images, extracting proposal crops, and labeling them based on how well they overlap with the known synthetic insect locations. Crops that overlap significantly with a known synthetic insect become positive training examples, while crops that don't overlap become negative examples.


The training is "lazy" - it doesn't happen when you create the classifier object, it happens the first time you try to use it for scoring. This is why there's a brief delay the first time you run the pipeline, but subsequent runs are fast because they load the cached model.


The RF trains from \textbf{synthetic image ground truth}. Proposal crops with
IoU\,$\geq$\,0.4 against a known synthetic GT box become positives; crops with
IoU\,$<$\,0.1 become hard negatives. This ensures the training distribution
matches exactly what the region proposer produces at inference time.

\subsection{Discussion}


\subsubsection{Cold-Start Strategy}

\textbf{Phase 0 (now):} HOG+LBP+RF trained on synthetic GT. P\,85\,\%,
R\,77\,\%, F1\,81\,\%. Sufficient for labeling-assist use.

\textbf{Phase 1:} Collect SLF reference photographs from iNaturalist
(CC-licensed, species ID 49837). Use real cutouts as sprites in the synthetic
generator to reduce the synthetic-to-real gap.

\textbf{Phase 2:} Active labeling loop --- run pipeline on real trap images,
route uncertain candidates to Label Studio, experts confirm/reject.
Target: 50 confirmed real positives, 150 hard negatives.

\textbf{Phase 3:} Call \code{DescriptorClassifier.train\_supervised()} with
confirmed real crops. RF retrains in under a minute. With $\geq$500 real
positives, fine-tune YOLOv8-small end-to-end.

\subsubsection{Assumptions and Limitations}

\textbf{Synthetic evaluation only.}
The 85\,\%/77\,\% figures are on synthetic test images. Real-world performance
will differ until the classifier is retrained on confirmed real positives.

\textbf{Overlapping insects.}
Adjacent SLF adults may be counted as one detection. 

\textbf{No temporal context.}
The pipeline processes images independently. Trap-level metadata (location,
date, age) could improve precision via domain-constrained priors.

\textbf{HOG sensitivity to image quality.}
HOG degrades with severe blur or very low resolution. Images should be at
least 800\,$\times$\,600\,px with the trap board filling most of the frame.



\subsubsection{Why not CLIP?}
The initial attempt used OpenAI's CLIP model for zero-shot classification via text prompts. However, we discovered it has 
two practical drawbacks that inhibit its utility given our constraints:
\begin{enumerate}
    \item It requires a GPU — a requirement unavailable in the current deployment.
    \item In the absence of those dependencies it falls back to a neutral stub, producing 0.5 scores for every candidate and reducing the classifier to the heuristic filter alone (measured: <38\% precision).
\end{enumerate}

CLIP remains the intended upgrade path once GPU infrastructure is available, as its visual representations are expected to generalise 
better across trap manufacturers, lighting conditions, and geographic variation in SLF appearance.

\subsection{Component descriptions}

\textbf{Ingestion service.}
A thin stateless HTTP service (FastAPI or similar) that validates image uploads,
attaches metadata (trap ID, GPS, timestamp), writes the raw file to object
storage (S3/GCS), and pushes a job reference to a message queue. No ML
dependencies; independently scalable.

\textbf{Detection worker.}
A containerised Python process that pulls jobs, loads images, and runs the
four-stage pipeline. The RF classifier is exported to ONNX at deployment time,
removing the scikit-learn runtime dependency and reducing per-image inference
time. Results are written to the database tagged with the model version,
enabling retrospective analysis when the model changes.

\textbf{Review queue and annotation tool.}
Detections with a final score in the \textbf{uncertainty band 0.45--0.70}
(just around the classification boundary) are routed to a review queue rather
than silently accepted or rejected. These are precisely the cases where the
model is least confident and where expert labels are most informative. Label
Studio provides a confirm/reject interface; an expert can typically process
8--12 uncertain crops in five minutes. The pipeline already exports
pre-annotated Label Studio tasks via \code{scripts/annotate\_helper.py}.

\textbf{Training data store.}
A versioned directory structure (or DVC-tracked dataset) keeping confirmed
positives, confirmed negatives, and synthetic images in separate partitions.
This allows the retraining job to adjust the mixing ratio as real-label counts
grow: early runs are dominated by synthetics; later runs are dominated by
confirmed real examples.

\textbf{Retraining job.}
Triggered when $\geq$50 new confirmed labels have accumulated since the last
retraining. The job: (1)~merges new and existing data, (2)~retrains the RF
or fine-tunes YOLOv8 depending on the label count, (3)~evaluates against the
held-out validation set, (4)~gates deployment --- the new model is only
promoted if both precision and recall meet or exceed the current model, (5)~
exports to ONNX, and (6)~deploys to the detection worker.

\subsection{Feedback loop dynamics}

The loop has a self-improving character. As more images are processed, more
uncertain candidates accumulate; confirmed labels produce a better model; a
better model is more decisive, producing fewer uncertain candidates per image.
The loop converges toward a stable operating point where expert review effort
decreases over time (Figure~\ref{fig:feedbackloop}).

Three failure modes can break this loop and should be monitored:

\begin{enumerate}[leftmargin=1.6em, itemsep=3pt]
  \item \textbf{Label starvation} --- the review queue fills faster than
        experts can clear it. Mitigation: raise the detection threshold
        temporarily (fewer detections, fewer reviews) or onboard additional
        reviewers.
  \item \textbf{Concept drift} --- a new trap type or change in field
        photography shifts the image distribution so that confident predictions
        start degrading. Mitigation: monitor the fraction of detections falling
        into the uncertainty band over time; a rising trend signals drift and
        triggers a targeted re-labeling campaign.
  \item \textbf{Annotation fatigue} --- reviewer accuracy drops in large
        batches. Mitigation: cap sessions at 30 minutes; use inter-annotator
        agreement on a 5\,\% overlap sample to detect label quality drift.
\end{enumerate}









\section{Conclusion}
This work presents a PoC computer vision pipeline for detecting and counting Spotted Lanternfly adults in pheromone trap photographs, designed to address the critical challenge of having no existing labeled training dataset. The four-stage pipeline combines classical image processing (CLAHE preprocessing, MSER+Canny region proposal) with a heuristic color and shape filter, followed by a HOG+LBP+Random Forest descriptor classifier trained entirely from synthetic ground truth data. This approach
achieves 80\% precision and 83\% recall on synthetic test images while requiring no GPU, no internet access, and no manually labeled real images—making it immediately deployable in resource-constrained field environments. The system includes a production architecture with a feedback loop that routes uncertain detections (scores 0.45-0.70) to expert review, enabling continuous improvement through active learning as confirmed labels accumulate, with a clear upgrade path to modern CNN-based detectors once sufficient real training data and GPU infrastructure become available. While performance on real trap images remains to be validated, this classical computer vision baseline provides a practical foundation for automated SLF monitoring that can begin generating useful data immediately while bootstrapping the collection of labeled examples needed for more sophisticated approaches
